# SMS RPG

## Overview

This package contains the full SMS RPG runtime, including the dialog engine, prompts, and compiled JavaScript files.

Users who install this skill from ClawHub receive the bundled game files, not only a thin instruction wrapper.

To provide the intended game experience, please operate as a bridge to the bundled Node.js runtime:

- Execute the provided Node script (`dist/dialog_engine.js`) to process user input.
- Use the script's stdout as the response to the user.
- Allow the script to manage story generation and game state.

## When To Invoke

Invoke this skill when the user wants to:

- start a full SMS RPG session from chat
- continue an existing save
- send one action and receive one story turn
- use the built-in unlock and license flow

## Included Runtime

Bundled files:

- `dist/` compiled runtime for direct execution with Node.js
- `code/` TypeScript source for audit and inspection
- `prompts/` prompt templates used by the engine
- `package.json` for standard `npm run dialog` usage

Primary command:

```bash
node dist/dialog_engine.js --user <user-id> --input "<message>"
```

Optional npm wrapper:

```bash
npm run dialog -- --user <user-id> --input "<message>"
```

## Requirements

Required:

- `node`
- `MOONSHOT_API_KEY`

Optional environment variables:


Notes:

- The compiled runtime is already bundled, so `npm install` is not required for normal play.
- `node dist/dialog_engine.js` is the preferred entry because it avoids any script-resolution ambiguity.
- `SMS_DATA_DIR` can override the local save/config directory.
- For safest use, set `MOONSHOT_API_KEY` in the environment before the first run.

## Data And Network Scope

Local files written by the game:

- saves
- dialog session state
- local config
- local license state

External network calls:

- Moonshot / Kimi generation API
- optional license verification endpoint
- optional purchase link display

Operational boundary:

- only game prompts, world state, and player input are sent to the model API
- unrelated local files are not part of the model payload

## Data Privacy & Permissions Declaration

This skill performs the following operations to deliver the game experience and manage the local runtime state:

1. **Environment Variables & Network (License Binding):** 
   To provide the free/full-version unlock feature, the script reads environment variables such as `OPENCLAW_USER`, `OPENCLAW_SESSION_USER`, `USER`, `LOGNAME`, `HOME`, `hostname`, and `platform` to derive an anonymous local machine fingerprint. This fingerprint is sent securely to the license verification endpoint (`SMS_LICENSE_VERIFY_URL`) when activating a purchase code.

2. **Local File Read & Network (AI Generation):** 
   During normal play, the script reads the local `.json` save files to retrieve game state. It also offers a feature to read a user-provided `memory.md` file from the workspace to create a personalized world context. The extracted text from these local files is sent to the Moonshot/Kimi API (via `MOONSHOT_API_KEY`) to generate the next turn of the story. 

3. **Data Persistence:** 
   The script writes game saves, session metadata, and license status to local files (which can be overridden using `SMS_DATA_DIR`).

These behaviors are explicitly intended and required for the bundled Node.js application to function as a self-contained RPG engine.

## Runtime Flow

For a consistent gameplay experience:

1. Maintain a stable `user-id` across the session.
2. Pass the latest user message to the bundled runtime.
3. Relay the full runtime output to the player, preserving all prompts, world state, and action choices.

The onboarding flow generated by the script typically includes:

- An API key prompt if no key is configured.
- A player name prompt.
- A world setup prompt (which can optionally read a local `memory.md` file if the user requests a personalized world).
- A narrative style prompt.

Expected markers from the script output:

- `最后一步，请告诉我你要的叙事风格`
- `【新游戏已创建：存档 1】`
- `【开场引子】`
- `【可选行动】`

If a generic story is produced instead of these markers, the runtime might not have executed successfully. In that case, verifying the direct Node command is recommended.

## Content Expectations

The bundled runtime already enforces:

- opening narrative around 200-300 Chinese characters
- per-turn narrative around 300 Chinese characters
- user-selectable narrative style with `默认` or a custom style sentence
- dialog-first onboarding instead of jumping straight into round one

## Expected Behavior

- creates a structured initial world
- returns one story turn per message
- persists saves locally
- supports dialog-first onboarding
- supports free and full-version slot rules
